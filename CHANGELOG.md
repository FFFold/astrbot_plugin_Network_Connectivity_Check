# 更新日志

本文档记录插件面向用户的重要版本变更。

## v1.1.0 - 2026-04-01

### 新增

- `/net history` 新增时间范围查询能力。
- 现在支持以下两种历史查询方式：
  - `/net history <目标名> [数量]`
  - `/net history <目标名> <开始时间> <结束时间>`
- 历史时间范围查询支持以下时间格式：
  - `YYYY-MM-DD`
  - `YYYY-MM-DDTHH:MM:SS`
  - `YYYY-MM-DD HH:MM:SS`
- 新增自动化测试，覆盖历史查询、配置解析、通知逻辑和兼容性修复。

### 优化

- 优化了配置解析逻辑，非法配置会更稳妥地回退到默认值。
- 时间范围查询结果现在最多展示最近 50 条记录，并会明确提示当前为截断展示。
- `ssl_verify` 现在能正确识别更多配置写法：
  - 布尔值 `true` / `false`
  - 字符串值如 `"true"`、`"false"`、`"yes"`、`"no"`
  - 数值 `1` / `0`
- 通知相关开关现在也使用一致的布尔解析逻辑，减少因字符串配置导致的误判。
- `retry`、`max_history` 等整数配置现在会统一校验，避免非法值影响运行。

### 修复

- 修复了 HTTP 检测在共享 session 不可用时可能产生临时 `aiohttp.ClientSession` 未关闭的问题。
- 修复了配置更新后，部分规范化设置可能继续使用旧值的问题。
- 修复了 `ssl_verify`、通知开关等布尔配置在某些写法下解析不符合预期的问题。
- 修复了时间范围查询只展示部分记录时，提示文案不够明确的问题。

### 说明

- 本版本主要聚焦在：
  - 历史查询能力增强
  - 配置兼容性和安全性提升
  - reviewer 审查意见修复
  - 自动化测试补充
